Atlassian User Interface

AUI is a tailor-made frontend library for creating a user interface according to the Atlassian Design Guidelines

Getting started

Most Atlassian products have AUI installed as a plugin. The flat pack is great for prototyping or deploying in apps which can't load a plugin.

Documentation

AUI provides JavaScript, CSS and Soy templates (or markup patterns) for a range of ready-made UI components.

Serve AUI from CDN

Serving AUI from a CDN will make your pages load faster and you won't have to maintain out of date resources.

To use core AUI components, include the following CSS and JS in your HTML:

<link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/css/aui.css" media="all">
<!--[if lt IE 9]><link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/css/aui-ie.css" media="all"><![endif]-->
<!--[if IE 9]><link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/css/aui-ie9.css" media="all"><![endif]-->
<script src="//ajax.googleapis.com/ajax/libs/jquery/1.8.3/jquery.min.js"></script>
<script src="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/js/aui.js"></script>
<!--[if lt IE 9]><script src="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/js/aui-ie.js"></script><![endif]-->

To use additional components, include the following resources:

Date picker

<script src="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/js/aui-datepicker.js"></script>

Experimental components

<link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/css/aui-experimental.css" media="all">
<script src="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/js/aui-experimental.js"></script>

Javascript Soy templates

<script src="//cdnjs.cloudflare.com/ajax/libs/aui/5.2/js/aui-soy.js"></script>

How to load this flat pack

If you want everything including AUI's dependencies, Experimental features and compiled-JS Soy templates, add this to the HEAD of your templates or page:

<link rel="stylesheet" href="aui/css/aui-all.css" media="all">
<!--[if lt IE 9]><link rel="stylesheet" href="aui/css/aui-ie.css" media="all"><![endif]-->
<!--[if IE 9]><link rel="stylesheet" href="aui/css/aui-ie9.css" media="all"><![endif]-->
<script src="aui/js/aui-all.js"></script>
<!--[if lt IE 9]><script src="aui/js/aui-ie.js"></script><![endif]-->

...remembering to update the path to point to the location of your copy. This is the recommended/default option.

For more granular control (eg. to skip Experimental or drop IE8 support), you can use the appropriate combination of these lines (note the dependencies are required):

<link rel="stylesheet" href="aui/css/aui.css" media="all">
<link rel="stylesheet" href="aui/css/aui-experimental.css" media="all">
<!--[if lt IE 9]><link rel="stylesheet" href="aui/css/aui-ie.css" media="all"><![endif]-->
<!--[if IE 9]><link rel="stylesheet" href="aui/css/aui-ie9.css" media="all"><![endif]-->
<script src="aui/js/aui-dependencies.js"></script>
<script src="aui/js/aui.js"></script>
<script src="aui/js/aui-experimental.js"></script>
<script src="aui/js/aui-soy.js"></script>
<!--[if lt IE 9]><script src="aui/js/aui-ie.js"></script><![endif]-->
